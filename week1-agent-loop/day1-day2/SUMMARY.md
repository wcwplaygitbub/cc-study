## 二、核心理解

### Agent Loop 的一次完整迭代

用户信息发来的时候，组合系统提示词 + 上下文 + 加工具开始发送给 LLM，LLM 返回如果是工具调用，则进行工具调用，把结果也合入上下文，再次发给大模型，一直循环到没有工具调用只有文本输出，就是一次完整 loop。

```
用户输入 → 构建请求（系统提示+上下文+工具）→ LLM
    ↓
有 tool_use → 执行工具 → 结果加回上下文 → 继续发给 LLM
    ↓
没有 tool_use → 只有文本 → 返回给用户 → 循环结束
```

### 消息列表在循环中的变化

初始只有用户消息；每次 LLM 回复若有 tool_use，执行后工具结果作为 user 角色加回消息列表；若没有工具则循环结束，AI 最终文字回复也加回消息列表。随着对话进行，消息列表不断增长。

### tool_use 块 vs 普通文字

tool_use 有标签 `type: 'tool_use'`，包含 `name`（工具名）、`input`（参数）、`id`（唯一标识）；普通文字/思考是 context 类型（`text` 或 `thinking`），直接展示给用户。

### 上下文压缩的理解

当对话越来越长，消息列表不断膨胀，快要接近 LLM 上下文窗口上限时（超过 80% 安全阈值），触发 AutoCompact，用 LLM 把 boundary 之前的历史消息压缩成摘要，释放 token 空间。压缩不可逆，是单向的。

### 循环退出条件

LLM 返回的流式内容中，没有 `tool_use` 块，只有 `text` 或 `thinking` 类型 → 循环退出，返回最终文字回复给用户。
